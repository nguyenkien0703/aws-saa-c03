## Content

Docker Compose là một công cụ mạnh mẽ giúp bạn quản lý nhiều container dễ dàng bằng cách sử dụng một file YAML duy nhất. Điều này đặc biệt hữu ích khi bạn cần chạy các ứng dụng phức tạp bao gồm nhiều thành phần như **web server**, **database**, **queue**, hoặc **cache**.

Bài viết này sẽ hướng dẫn bạn từng bước cách cài đặt, sử dụng, tối ưu Docker Compose để triển khai một hệ thống nhanh chóng, linh hoạt và dễ quản lý.

---

## 1. Docker Compose Là Gì?

Docker Compose là một công cụ giúp bạn định nghĩa và quản lý nhiều container trong một ứng dụng bằng cách sử dụng một file **`docker-compose.yml`**.

Thay vì chạy từng container riêng lẻ với nhiều câu lệnh **`docker run`**, Docker Compose cho phép bạn chạy tất cả container chỉ với một câu lệnh duy nhất.

Lợi ích của Docker Compose:

- **Triển khai nhanh chóng**: Chỉ cần một file YAML để định nghĩa toàn bộ ứng dụng.
- **Dễ dàng quản lý**: Gộp tất cả container vào một cấu hình chung.
- **Tự động hóa**: Có thể chạy, dừng, hoặc restart toàn bộ hệ thống một cách dễ dàng.

Docker Compose thường được sử dụng trong môi trường phát triển, testing hoặc staging, nhưng cũng có thể áp dụng trong production với các kỹ thuật bổ sung.

---

## 2. Cài Đặt Docker Compose

Trước khi sử dụng Docker Compose, hãy đảm bảo bạn đã cài Docker trên máy.

### **Cài đặt Docker (nếu chưa có)**

Nếu bạn chưa cài Docker, có thể cài đặt bằng lệnh:

```bash
sudo apt update
sudo apt install -y docker.io
sudo systemctl enable --now docker
```

### **Cài đặt Docker Compose**

Với Ubuntu/Debian, chạy lệnh sau:

```bash
sudo apt install -y docker-compose
```

Kiểm tra phiên bản:

```bash
docker-compose --version
```

---

## 3. Tạo File `docker-compose.yml`

Bây giờ, hãy tạo một ứng dụng web đơn giản gồm:

- **Nginx** làm web server
- **MySQL** làm database

Tạo file `docker-compose.yml`:

```yaml
version: '3.8'
services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
    depends_on:
      - db

  db:
    image: mysql:5.7
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: mydb
      MYSQL_USER: user
      MYSQL_PASSWORD: password
```

File này định nghĩa hai dịch vụ:

- **`web`**: Chạy Nginx trên port 8080.
- **`db`**: Chạy MySQL với thông tin đăng nhập sẵn.

---

## 4. Chạy Ứng Dụng Với Docker Compose

Bây giờ, chúng ta có thể khởi động toàn bộ ứng dụng với một lệnh duy nhất:

```bash
docker-compose up -d
```

Thao tác này sẽ:

- Tải về các image cần thiết (nếu chưa có).
- Tạo và chạy container.
- Liên kết container theo đúng thiết lập trong file YAML.

Kiểm tra container đang chạy:

```bash
docker-compose ps
```

Bạn có thể truy cập **Nginx** tại `http://localhost:8080`.

---

## 5. Tối Ưu Hóa Docker Compose Cho Hiệu Suất Cao

### **1. Sử dụng Docker Volumes thay vì Bind Mounts**

Thay vì sử dụng bind mount (`./html:/usr/share/nginx/html`), bạn nên dùng volumes để tránh lỗi quyền truy cập và tăng hiệu suất:

```yaml
volumes:
  nginx_data:

services:
  web:
    volumes:
      - nginx_data:/usr/share/nginx/html
```

### **2. Tối ưu Network Giữa Các Container**

Docker Compose tự động tạo một network riêng, nhưng bạn có thể định nghĩa nó rõ ràng hơn để tối ưu:

```yaml
networks:
  my_network:

services:
  web:
    networks:
      - my_network

  db:
    networks:
      - my_network
```

Điều này giúp dễ kiểm soát luồng dữ liệu giữa các container.

### **3. Sử dụng `.env` File Để Quản Lý Biến Môi Trường**

Thay vì hardcode thông tin database trong YAML, bạn nên tách chúng ra file `.env`:

**Tạo file `.env`:**

```env
MYSQL_ROOT_PASSWORD=rootpass
MYSQL_DATABASE=mydb
MYSQL_USER=user
MYSQL_PASSWORD=password
```

**Sửa file `docker-compose.yml`:**

```yaml
environment:
  MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
  MYSQL_DATABASE: ${MYSQL_DATABASE}
  MYSQL_USER: ${MYSQL_USER}
  MYSQL_PASSWORD: ${MYSQL_PASSWORD}
```

Cách này giúp bảo mật và dễ dàng thay đổi cấu hình.

---

## 6. Dừng Và Xóa Container

Để dừng tất cả container, chạy:

```bash
docker-compose down
```

Nếu muốn xóa toàn bộ dữ liệu, thêm `-v`:

```bash
docker-compose down -v
```

---

## 7. Kết Luận

Docker Compose giúp bạn triển khai nhiều container một cách dễ dàng chỉ với một file YAML duy nhất. Nếu bạn làm việc với các hệ thống phức tạp, đây là công cụ không thể thiếu.

Những tối ưu trên giúp bạn có một hệ thống chạy nhanh hơn, bảo mật hơn và dễ quản lý hơn trong cả môi trường development lẫn production.

Hy vọng hướng dẫn này giúp bạn hiểu rõ hơn về Docker Compose và có thể áp dụng vào dự án thực tế!

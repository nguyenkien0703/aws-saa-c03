Jenkins là một công cụ CI/CD rất phổ biến và được nhiều người sử dụng.

## Chuẩn bị

Trước khi ta tiến hành viết code cho Terraform và tệp tin CI/CD, ta sẽ cần chuẩn bị các thứ sau để Terraform có thể chạy được trong Jenkins.

### Github

Ở bài này mình chỉ hướng dẫn xài Jenkins với Terraform thế nào. Chi tiết hơn các bạn có thể đọc ở bài này [How To Set Up Continuous Integration With Git and Jenkins?](https://www.lambdatest.com/blog/how-to-setup-continuous-integration-with-git-jenkins).

Ta tạo một Github Repository (chế độ *Public*) để chứa source code và kết nối tới Jenkins. Source code sử dụng ở bài này [Terraform Jenkins Example](https://github.com/hoalongnatsu/terraform-series-jenkins-example). Tiếp theo tạo *Project* ở trên Jenkins với tên `terraform-jenkins`, chọn kiểu Pipeline.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/d8af0337-f00d-49ee-8ae3-59b93c51b454/Untitled/w=1920,quality=90,fit=scale-down)

Bấm qua mục Pipeline và điền vào như bên dưới:

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/45b35ad4-c0e8-4099-9e23-44bea4c97caf/Untitled/w=1920,quality=90,fit=scale-down)

Bấm lưu.

### AWS Credentials

Vì ta xài Terraform với AWS nên ta cần cấu hình AWS Credentials ở trên Jenkins cho Terraform. Theo hướng dẫn này để tạo IAM user với quyền AdministratorAccess: [Creating your first IAM admin user and user group](https://docs.aws.amazon.com/IAM/latest/UserGuide/getting-started_create-admin-group.html).

Sau đó tạo Secret Key cho IAM User ở trên. Ở mục Jenkins chọn **Manage Jenkins -> Manage Credentials**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/03928232-5981-4bc0-9cbc-a99fe0f25516/Untitled/w=1920,quality=90,fit=scale-down)

Ở mục **Stores scoped to Jenkins** chọn Jenkins.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/a25f85c0-d1c1-4481-8969-165a564f8297/Untitled/w=1920,quality=90,fit=scale-down)

Chọn **Global credentials**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/2990ab7c-b52e-49ba-9f5b-4d2a22c1f45c/Untitled/w=1920,quality=90,fit=scale-down)

Chọn **Add Credentials**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/2c8a32e0-1db0-4ac3-bf78-a79595a61e5f/Untitled/w=1920,quality=90,fit=scale-down)

1. Ở trường Kind chọn Secret Text
2. Trường Scope để mặc định
3. Trường Secret điền vào giá trị của của `AWS_ACCESS_KEY_ID`
4. Trường ID là tên của Secret, đặt tên cho nó là `aws-secret-key-id`
5. Trường Description nhập gì cũng được

Tương tự tạo **Secret text** khác để chứa giá trị của `AWS_SECRET_ACCESS_KEY` và đặt tên cho nó là `aws-secret-access-key`.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/d8a47078-926b-4f00-b96a-671d7d36ba61/Untitled/w=1920,quality=90,fit=scale-down)

### S3 Backend

Tiếp theo ta sẽ tạo S3 Backend để lưu trữ Terraform State, xem kĩ hơn về S3 Backend ở này này [Sử dụng S3 Standard Backend vào dự án](/terraform-series/terraform/bai-8-su-dung-s3-standard-backend-vao-du-an). Tải Repo sau về [Terraform Series](https://github.com/hoalongnatsu/terraform-series), di chuyển tới `bai-14/s3-backend` và chạy các câu lệnh sau:

Copy

```
terraform init
terraform apply -auto-approve
```

Khi terraform chạy xong các giá trị của S3 Backend sẽ được hiển thị.

Sao chép lại các giá trị này 😁.

### Kết hợp với Terraform

Để chạy Terraform trong Jenkins ta có các cách sau:

1. Cài Terraform lên trên Build Agent
2. Sử dụng Docker Container
3. Sử dụng Terraform Plugin

Ta sẽ sử dụng cách thứ ba trong bài này, di chuyển tới **Manage Jenkins -> Manage Plugins** tìm Terraform Plugin và bấm Install.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/371e4bdf-c84e-44c3-b5b5-fada93ef1bcb/Untitled/w=1920,quality=90,fit=scale-down)

Sau đó truy cập **Manage jenkins -> Global Tool Configuration**, kiếm phần Terraform và cấu hình nó như sau:

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/269ee3ff-5400-43ed-8904-0dd44b06fab2/Untitled/w=1920,quality=90,fit=scale-down)

Tiếp theo ta tiến hành xây dựng luồng CI/CD.

## CI/CD

Tạo một tệp tin tên là `main.tf`.

Copy

Cấu hình S3 Backend như sau:

Copy

```
terraform {
  backend "s3" {
    bucket         = "terraform-series-s3-backend"
    key            = "terraform-jenkins"
    region         = "us-west-2"
    encrypt        = true
    role_arn       = "arn:aws:iam::<ACCOUNT_ID>:role/Terraform-SeriesS3BackendRole"
    dynamodb_table = "terraform-series-s3-backend"
  }
}
```

**Sửa lại** `<ACCOUNT_ID>` **với ID của bạn**. Tiếp theo ta tạo `Jenkinsfile`:

Copy

Để sử dụng Terraform trong Pipeline thì ta dùng thuộc tính:

Copy

```
tools {
  terraform 'terraform'
}
```

Đây là Global Tool ta đã cấu hình ở trên khi cài Terraform Plugin. Và để sử dụng AWS Credentials ta dùng hai thộc tính:

Copy

```
environment {
  AWS_ACCESS_KEY_ID     = credentials('aws-secret-key-id')
  AWS_SECRET_ACCESS_KEY = credentials('aws-secret-access-key')
}
```

Hàm Credentials được dùng để lấy giá trị của **Secret Text** mà ta đã tạo. Các đoạn code còn lại là để chạy Terraform, gồm có bước `init`, `plan` và `deploy`.

Đoạn code phần `deploy` sẽ khác một chút, nó có thêm thuộc tính `input`:

Copy

```
input {
  message "Do you want to proceed for production deployment?"
}
```

Ta sử dụng thuộc tính này để làm tính năng Manual Approve. Ta sẽ không cho CI/CD tự động chạy `terraform apply` mà ta cần xem trước các resource nào sẽ được tạo ra. Ta đẩy code lên Github, truy cập vào Jenkins để thực thi CI/CD. Đây là giao diện pipeline cũ của Jenkins.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/654705d2-e72d-499b-8547-261fd36de7ce/Untitled/w=1920,quality=90,fit=scale-down)

Bấm vào **Open Blue Ocean** để chuyển qua giao diện mới vì nó đẹp hơn, các bạn có thể dùng giao diện cũ nếu chưa quen.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/fda7a306-79bc-465f-a6fc-f17815df1184/Untitled/w=1920,quality=90,fit=scale-down)

Bấm Run để nó thực thi Job.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/fa753d54-e4bb-4d77-b5af-7e1905c45631/Untitled/w=1920,quality=90,fit=scale-down)

Bấm vào Job để xem *logs*.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/c9387203-37ab-448d-9147-3c0fb4b2cfc1/Untitled/w=1920,quality=90,fit=scale-down)

Phần `Apply Resources` nó ở trạng thái chờ, sau khi xem phần `plan` xong thấy ổn thì ta bấm **Proceed**. Kiểm tra AWS Console bạn sẽ thấy EC2 đã được tạo ra bằng CI/CD thành công.

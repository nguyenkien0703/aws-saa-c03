## Cài đặt và cấu hình Git cơ bản

[Git](https://devops.vn/tag/git) là hệ thống quản lý phiên bản quan trọng đối với lập trình viên, hỗ trợ lưu trữ mã nguồn, làm việc nhóm và bảo toàn dữ liệu. Trước khi sử dụng, bạn cần thực hiện cài đặt và cấu hình các thông số cơ bản.

### 1. Cài đặt Git

Tùy vào hệ điều hành đang sử dụng, bạn thực hiện theo các hướng dẫn sau:

- **Windows**: Tải bộ cài tại [git-scm.com](https://git-scm.com/downloads) và tiến hành cài đặt.
- **macOS**: Sử dụng Homebrew với lệnh `brew install git` hoặc tải trực tiếp từ [trang chủ](https://git-scm.com/install/mac).
- **Linux (Ubuntu)**: Chạy các lệnh sau trong terminal:

```bash
sudo apt update
sudo apt install git -y
```

Sau khi cài đặt, hãy kiểm tra phiên bản để đảm bảo Git đã hoạt động:

```bash
git --version
```

### 2. Cấu hình thông tin người dùng

Thông tin này sẽ được lưu lại trong lịch sử các bản commit. Bạn cần thiết lập tên và email bằng hai lệnh sau:

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

- **user.name**: Tên hiển thị (Ví dụ: “Nguyen Van A”).
- **user.email**: Email liên kết với tài khoản GitHub hoặc GitLab.

Để kiểm tra lại các cấu hình đã thiết lập:

```bash
git config --list
```

### 3. Thiết lập Alias (Tùy chọn)

Để tối ưu hóa thời gian gõ lệnh, bạn có thể tạo các phím tắt (alias) cho những lệnh dài:

```bash
git config --global alias.co checkout
git config --global alias.ci commit
```

Sau khi thiết lập, bạn có thể dùng `git co` thay cho `git checkout` và `git ci` thay cho `git commit`.

### Một số lưu ý

- **Kiểm tra sau mỗi bước**: Luôn xác nhận lại lệnh vừa thực hiện để tránh lỗi tích tụ.
- **Sử dụng Terminal**: Làm quen với Git Bash (Windows) hoặc Terminal (macOS/Linux) là cách hiệu quả nhất để sử dụng Git.
- **Ghi chú**: Lưu lại các lệnh thường dùng để tra cứu nhanh khi cần thiết.

Tiếp theo chúng ta sẽ khởi tạo và quản lý một repository.

## Content

**Microsoft vừa thông báo Azure DevOps MCP Server đã chính thức kết thúc giai đoạn public preview và đi vào hoạt động. Đây là một “cầu nối” open-source chạy nội bộ, cho phép các AI agent như GitHub Copilot truy cập và tương tác an toàn với các artifact của Azure DevOps (như work items, pull requests) mà không làm lộ dữ liệu ra bên ngoài.**

![019a5ed1-bc3f-7620-883e-1d0d1fb67a73](https://devops.vn/uploads/images/2025/11/019a5ed1-bc3f-7620-883e-1d0d1fb67a73.webp)

Theo [blog Azure DevOps](https://dev.blogs.microsoft.com/devops/), MCP Server là một máy chủ chạy local hoặc bên trong mạng nội bộ của tổ chức. Vai trò của nó là cung cấp context phong phú từ Azure DevOps cho các trợ lý AI. Thay vì Copilot phải gọi ra các API bên ngoài, nó có thể tương tác trực tiếp với các artifact của Azure DevOps bao gồm **work items, pull requests, builds, test plans, và wikis** dưới các điều kiện bảo mật nghiêm ngặt.

Phiên bản GA này mang đến nhiều cải tiến đáng kể về **authen, authorization** và tooling, dựa trên phản hồi từ những người dùng sớm. Microsoft đã bổ sung “domain-scoping” (giới hạn phạm vi) cho các công cụ, tinh chỉnh luồng đăng nhập và tối ưu các API nội bộ.

Việc phát hành này được xem là một bước tiến quan trọng trong việc cung cấp khả năng **“agentic AI”** ngay trong IDE. Với MCP Server, GitHub Copilot có thể **hỏi hoặc thực thi các tác vụ** liên quan đến DevOps như tóm tắt tình trạng sprint, tạo/cập nhật work items, hoặc liên kết pull requests mà **không cần rời khỏi IDE** (Visual Studio hoặc VS Code).

Giải pháp này khác biệt với GitHub MCP Server. Trong khi GitHub MCP Server được thiết kế để kết nối Copilot với nền tảng SaaS của GitHub (tập trung vào repository, issues), thì Azure DevOps MCP Server lại **nhắm vào các enterprise** cần các cài đặt **on-premise** hoặc network-island, đảm bảo dữ liệu nhạy cảm của Azure DevOps không rời khỏi mạng nội bộ.

Main point: Use Application Load Balancer (ALB) with CloudFront over using ALB alone

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/cbe1b85e-4390-4974-9b69-80241a410b0b/FB_Post/w=1920,quality=90,fit=scale-down)

## **Reduced Latency**

CloudFront delivers content from edge locations closer to users and utilizes the AWS backbone network from the edge to the ALB.

ALB without CloudFront: load time =~ 750ms

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/f11cd124-e0d9-437f-9b02-035210b2aeb4/image/w=1920,quality=90,fit=scale-down)

ALB with CloudFront: load time =~ 440ms

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/77b40f70-c91b-4330-aa85-04613c079ebd/image/w=1920,quality=90,fit=scale-down)

## **Connection Optimization**

CloudFront supports TLS 1.3

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/e1e0f3b0-2cfe-4968-8061-fee7996e8934/image/w=1920,quality=90,fit=scale-down)

CloudFront maintains persistent connections to the ALB, which minimizes the overhead of establishing new TCP connections for each request.

ALB without CloudFront

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/b18409f7-5452-4bed-b6c1-573f9627d6f1/image/w=1920,quality=90,fit=scale-down)

ALB with CloudFront reduces initial connection: **278.8ms -> 17.5ms**

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/8b065838-108f-4236-9e8d-8afee2583d66/image/w=1920,quality=90,fit=scale-down)

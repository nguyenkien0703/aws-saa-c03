## Tổng quan

Hệ thống Log gồm ba thành phần chính:

- FluentBit thu thập logs và gửi logs tới Loki
- Loki là nơi lưu trữ logs
- Để xem logs ta dùng Grafana

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/3773126d-0e63-4a2f-a732-d4ef7af4b9de/Kubernetes_co_ban/w=1920,quality=90,fit=scale-down)

Ta dùng Helm để cài các công cụ trên lên Kubernetes.

## Loki

Tương tự như Prometheus dùng để lưu trữ metrics thì [Loki](https://grafana.com/docs/loki/latest/) dùng để lưu trữ logs.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/f3fe21f5-6e4e-4848-86b2-2119b43cd380/Untitled/w=1920,quality=90,fit=scale-down)

Add Grafana Repo cho Loki:

Copy

```
helm repo add grafana https://grafana.github.io/helm-charts
```

Tạo tệp tin `values.yaml`:

Copy

Ở trên ta cài Loki với cấu hình đơn giản cho môi trường dev, để triển khai cho môi trường product ta tham khảo ở đây [**Configure storage**](https://grafana.com/docs/loki/latest/setup/install/helm/configure-storage/). Cài Loki:

Copy

```
helm -n observability install --create-namespace -f values.yaml loki grafana/loki
```

Kiểm tra:

Copy

```
kubectl get pod -n observability

NAME                            READY   STATUS    RESTARTS   AGE
loki-0                          1/1     Running   0          50s
loki-gateway-18fi5ds94f-tgf4p   1/1     Running   0          50s
```

## FluentBit

Thêm Fluent Repo:

Copy

```
helm repo add fluent https://fluent.github.io/helm-charts
```

Tạo tệp tin `values.yaml` cho FluentBit:

Copy

Ở tệp tin trên các bạn chú ý thuộc tính `Labels {job="loki-collector"}`. Cài FluentBit:

Copy

```
helm -n observability install -f values.yaml fluent-bit fluent/fluent-bit
```

## Grafana

Để xem được logs ta cần cài Grafana:

Copy

```
helm -n observability install -f values.yaml grafana grafana/grafana
```

Kiểm tra:

Copy

```
kubectl get pod

NAME                            READY   STATUS    RESTARTS   AGE
fluent-bit-dt37a                1/1     Running   0          50s
grafana-537adka914-38dc1        1/1     Running   0          50s
loki-0                          1/1     Running   0          50s
loki-gateway-18fi5ds94f-tgf4p   1/1     Running   0          50s
```

Truy cập Grafana:

Copy

```
kubectl -n observability port-forward svc/grafana 3000:80
```

Lấy mật khẩu Grafana:

Copy

```
kubectl get secret -n observability grafana -o jsonpath="{.data.admin-password}" | base64 --decode ; echo
```

## Truy vấn logs

Mở trình duyệt và truy cập `localhost:3000` và nhập vào username là `admin` và password ở trên. Ở Menu ta chọn mục **Connections → Data sources → Add new data source** và chọn Loki. Điền vào như sau:

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/be539326-e037-450b-a57a-c79178e35151/Screenshot_2024-01-23_at_14.50.09/w=1920,quality=90,fit=scale-down)

Mở mục **HTTP headers** và thêm vào Header X-Scope-OrgID với Value là 1.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/d0efae79-c28c-4be3-b6b6-232684211527/Screenshot_2024-01-23_at_14.51.36/w=1920,quality=90,fit=scale-down)

Bấm Save. Sau đó ở Menu ta chọn mục **Explore** và điền vào giá trị `{job="loki-collector"}`.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/1aa0bc2b-c736-414a-ad3b-65a8e89b5ae7/Screenshot_2024-01-23_at_15.06.20/w=1920,quality=90,fit=scale-down)

Bấm **Run query** ta sẽ thấy logs.

## Kết luận

Nếu hệ thống chỉ cần lưu trữ logs để kiểm tra lỗi hoặc log real-time mà không cần phân tích phức tạp thì có thể sử dụng bộ Grafana Loki để hệ thống gọn nhẹ. Trường hợp cần phân tích log và query phức tạp thì nên dùng bộ EFK (Elasticsearch + FluentBit + Kibana).

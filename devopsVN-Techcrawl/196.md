Với Auto Scaling Group được đùng để tạo ra một nhóm EC2 chạy Web Server ở port 80. Relational Database Service dùng để lưu trữ dữ liệu. Và người dùng sẽ truy cập tới ứng dụng của ta thông qua Load Balancer. Đây là một mô hình rất phổ biến ở trên AWS, minh họa như sau.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/7b38d19e-d9d9-4b58-acec-cc85deed1f61/Untitled/w=1920,quality=90,fit=scale-down)

Ta sẽ có 3 thành phần chính trong mô hình trên là Networking, AutoScaling và Database. Từng thành phần chính sẽ được nhóm lại thành một Module như sau.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/a672e5b4-1c22-44be-8616-09643a82fef9/Untitled/w=1920,quality=90,fit=scale-down)

Ta sẽ viết Module cho Networking, AutoScaling và RDS. Tất cả các Module đều có quan hệ với nhau theo mô hình cây, mà thằng trên đầu được gọi là Root Module.

## Root Module

Tất cả các Workspace đều có một thằng được gọi là Root Module. Ở trong Root Module đó, chúng ta có thể có một hoặc nhiều Module con. Module có thể là Local Module với code nằm ở dưới máy của ta hoặc Remote Moudle, Moudle mà được để trên mạng và ta tải xuống bằng câu lệnh `terraform init`. Mô hình cây mối quan hệ giữ các Moudle.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/5780a3dc-c3e4-4377-9d2a-0c45d4aab0fe/Untitled/w=1920,quality=90,fit=scale-down)

Như ta thấy ở Moudle hình trên thì Networking, AutoScaling và RDS là Moudle con của Root Module. Và trong một Moudle nó có thể có chứa một hoặc nhiều Moudle khác, như Networking nó chứa VPC Moudle và SG (Security Group) Moudle, nếu một Moudle nằm trong một Moudle khác, ta gọi nó là Nested Moudle.

## Viết code

Giờ ta sẽ tiến hành viết code, ta tạo thư mục như sau.

Copy

```
.
├── main.tf
└── modules
    ├── autoscaling
    │   ├── main.tf
    │   ├── outputs.tf
    │   └── variables.tf
    ├── database
    │   ├── main.tf
    │   ├── outputs.tf
    │   └── variables.tf
    └── networking
        ├── main.tf
        ├── outputs.tf
        └── variables.tf
```

Ở file `main.tf` của Root, ta thêm vào đoạn code sau.

Copy

```
locals {
  project = "terraform-series"
}

provider "aws" {
  region = "us-west-2"
}

module "networking" {
  source = "./modules/networking"
}

module "database" {
  source = "./modules/database"
}

module "autoscaling" {
  source = "./modules/autoscaling"
}
```

## Networking Module

Đầu tiên ta sẽ viết code cho Networking Module, khi viết Module thì ta cần định nghĩa giá trị đầu vào và đầu ra của Module, ta có thể định nghĩa từ đầu hoặc khi ta viết Module xong ta thấy ta cần giá trị nào mà động thì ta thêm vào cũng được không nhất thiết phải định nghĩa từ ban đầu. Networking Module của ta sẽ có giá trị đầu vào và đầu ra như sau.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/a0493872-9304-4c72-bdc8-d70662fa9799/Untitled/w=1920,quality=90,fit=scale-down)

Cập nhật file `variables.tf` của Module.

Copy

```
variable "project" {
  type    = string
}

variable "vpc_cidr" {
  type    = string
}

variable "private_subnets" {
  type    = list(string)
}

variable "public_subnets" {
  type    = list(string)
}

variable "database_subnets" {
  type    = list(string)
}
```

Tiếp theo cập nhật file `main.tf` của Module.

Copy

Đây là Remote Module mà ta sẽ dùng câu lệnh `terraform init` để tải xuống, Module này sẽ tạo VPC cho ta. Với các giá trị trên thì VPC của ta khi được tạo sẽ như thế này.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/737f2565-539f-46a3-9363-0e5a72fa6935/Untitled/w=1920,quality=90,fit=scale-down)

Tiếp theo ta sẽ tiến hành tạo Secutiry Group cho VPC của ta, Secutiry Group của ta phải cho phép 3 thằng sau:

1. Cho phép truy cập port 80 của ALB từ mọi nơi
2. Cho phép truy cập port 80 của các EC2 từ ALB
3. Cho phép truy cập port 5432 của RDS từ EC2

Ta thêm SG Rule vào.

Copy

Để các Module bên ngoài có thể truy cập được các giá trị của Module này, ta cần định nghĩa giá trị đầu ra cho nó. Cập nhật file `outputs.tf`.

Copy

```
output "vpc" {
  value = module.vpc
}

output "sg" {
  value = {
    lb = module.lb_sg.security_group.id
    web = module.web_sg.security_group.id
    db = module.db_sg.security_group.id
  }
}
```

## Giá trị đầu ra

Để truy cập giá trị của một Module, ta dùng cú pháp sau `module.<name>.<output_value>`, ví dụ để truy cập giá trị `lb_sg id` của Networking Module.

Copy

```
module.networking.sg.lb
```

**Nên nhớ** `module.<name>` **thì** `name` **là tên ta khai báo khi ta sử dụng Module chứ không phải tên thư mục của Module nhé.** Ví dụ:

Copy

```
module "networking" {
  source = "./modules/networking"
}

module.networking.sg.lb
```

Copy

```
module "nt" {
  source = "./modules/networking"
}

module.nt.sg.lb
```

Vậy là ta đã viết xong Module, ta sử dụng nó như sau. Cập lại file `main.tf` ngoài Root.

Copy

## Database Module

Tiếp theo ta sẽ viết code cho Database Module, input và output của Database Module.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/666600c2-83d3-4e54-93d5-53b686493201/Untitled/w=1920,quality=90,fit=scale-down)

Ở trên AWS khi ta tạo RDS yêu cầu ta cần phải có một Subnet Group trước, rồi RDS mới được triển khai lên trên Subnet Group đó.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/37407e8d-08e8-472c-a271-98bee70b7de3/Untitled/w=1920,quality=90,fit=scale-down)

Để tạo Subnet Group bằng Terraform thì ta sẽ xài `aws_db_subnet_group` resource, ví dụ.

Copy

```
resource "aws_db_subnet_group" "default" {
  name       = "main"
  subnet_ids = [aws_subnet.frontend.id, aws_subnet.backend.id]

  tags = {
    Name = "My DB subnet group"
  }
}
```

Ở trên khi ta xài Module VPC, thì nó đã tạo sẵn cho ta một thằng Subnet Group sẵn, nên ta mới cần truyền thằng VPC vào Module Database, để ta đỡ phải một thằng Subnet Group khác. Ta lấy giá trị Subnet Group ở trong Module vpc như sau `module.networking.vpc.database_subnet_group`. Giờ ta sẽ viết code cho Module, cập nhật tệp tin `variables.tf` trong Database Module.

Copy

```
variable "project" {
  type = string
}

variable "vpc" {
  type = any
}

variable "sg" {
  type = any
}
```

Tệp tin `main.tf`.

Copy

Để tạo RDS trên AWS thì ta sẽ dùng `aws_db_instance` resource, ở trên ta chỉ định Engine của RDS mà ta sẽ xài là PostgreSQL 12.7, với 20GB lưu trữ, giá trị Subnet Group của RDS ta lấy giá trị truyền vào từ biến VPC. Mọi thứ có vẻ ổn, nhưng bạn để ý là ở trường `password`, hiện tại ta đang để cứng giá trị, nếu ta không muốn để cứng mà ta muốn giá trị này sẽ là động thì sao?

Ta sẽ dùng một resource khác trong Terraform giúp ta tạo Password động, sau đó ta sẽ truyền giá trị Password này vào Database, cập nhật code lại.

Copy

**Lưu ý khi ta sử dụng resource này thì Password của ta sẽ được lưu ở trong tệp tin State, lúc này ai truy cập vào tệp tin State cũng có thể thấy được Password, dẫn đến việc bảo mật của ta không được tốt, ta sẽ bàn về vấn đề secutiry ở một bài khác.**

Ta xuất giá trị RDS ra ngoài để bên ngoài có thể truy cập được.

Copy

```
output "config" {
  value = {
    user     = aws_db_instance.database.username
    password = aws_db_instance.database.password
    database = aws_db_instance.database.name
    hostname = aws_db_instance.database.address
    port     = aws_db_instance.database.port
  }
}
```

Ya đã viết xong Database Module, ta cập nhật lại tệp tin `main.tf` ở Root để có thể sử dụng được Module.

Copy

Có một điểm ta cần nói là ở tệp tin khai báo biến của Database Module, hai giá trị là VPC với SG ta khai báo dữ liệu là `any`.

Copy

```
...
variable "vpc" {
  type = any
}

variable "sg" {
  type = any
}
```

Khi ta muốn truyền một giá trị mà ta không biết nó thuộc loại dữ liệu nào thì ta sẽ khai báo kiểu dữ của nó là `any`.

## Autoscaling Module

Module cuối cùng mà ta sẽ viết là Autoscaling Module, đây là một Module chứa hơi nhiều thứ. Để tạo một Autoscaling Module trên AWS và khiến nó hoạt động được, ta cần một số Service phải tạo chung với nó như là Load Balancer, Launch Templates, … Trong khi Load Balancer, ta cũng cần phải tạo cho nó 3 thằng là Load Balancer + Target Group + LB Listener. Nên để tạo được ASG trên AWS ta sẽ dùng Module có sẵn thay vì phải tự viết code. Hình minh họa Autoscaling Module.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/8e9aa07a-edd7-49a9-830b-4bc4203ab952/Untitled/w=1920,quality=90,fit=scale-down)

Ta định nghĩa giá trị đầu vào và đầu ra của Autoscaling Module.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/15c82c9e-1ba6-4461-a2c0-5753ad0d24f9/Untitled/w=1920,quality=90,fit=scale-down)

Giờ ta sẽ tiến hành viết code, cập nhật tệp tin `variables.tf` của Autoscaling Module.

Copy

```
variable "project" {
  type = string
}

variable "vpc" {
  type = any
}

variable "sg" {
  type = any
}

variable "db_config" {
  type = object(
    {
      user     = string
      password = string
      database = string
      hostname = string
      port     = string
    }
  )
}
```

Tiếp theo ta sẽ khai báo ASG, để tạo được ASG thì ta cần có một Launch Templates đi kèm với nó, ASG sẽ dùng Template này để tạo EC2.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/913e8d36-8a63-40b7-8f1d-b65ecfd54a1c/Untitled/w=1920,quality=90,fit=scale-down)

Để tạo Launch Templates ta dùng `aws_launch_template` resource, cập nhật tệp tin `main.tf` của Autoscaling Module.

Copy

Tệp tin `run.sh`.

Copy

```
#!/bin/bash
yum update -y
yum install -y httpd.x86_64
systemctl start httpd
systemctl enable http
echo "$(curl http://169.254.169.254/latest/meta-data/local-ipv4)" > /var/www/html/index.html
```

Ở trên ta dùng `aws_ami` để lọc lấy ra Image ID của OS `amazon-linux-2`, sau đó gán ID này vào Launch Template, thuộc tính `user_data` ta định nghĩa đoạn code sẽ chạy khi EC2 của ta được tạo ra. Tiếp theo ta gán nó vào Autoscaling Group.

Copy

```
...
resource "aws_autoscaling_group" "web" {
  name                = "${var.project}-asg"
  min_size            = 1
  max_size            = 3
  vpc_zone_identifier = var.vpc.private_subnets

  launch_template {
    id      = aws_launch_template.web.id
    version = aws_launch_template.web.latest_version
  }
}
```

Tiếp theo vì RDS của ta được tạo ở chế độ Private, nên để EC2 có thể truy cập được tới DB ta phải gán IAM Role vào trong EC2 này, ở trong Terraform ta có thể cấu hinh2 nó thông qua thuộc tính `iam_instance_profile` của `aws_launch_template` resource. Ta cập nhật lại code như sau.

Copy

Ta dùng Module `terraform-in-action/iip/aws` để tạo `role` với toàn bộ quyền truy cập tới `logs` và RDS, sau đó ta gán náo vào `aws_launch_template`. Resource mà tiếp theo ta cần khai báo là Load Balancer, để cho phép người dùng truy cập được tới ASG của ta. Ta sẽ dùng `terraform-aws-modules/alb/aws`, thêm vào thêm `main.tf` đoạn code của LB.

Copy

Sau khi khai báo LB thì ta cập nhật lại thuộc tính `target_group_arns` của `aws_autoscaling_group` với giá trị `target_group_arns` được lấy ra từ Module LB. Cập nhật giá trị đầu ra của Module.

Copy

```
output "lb_dns" {
  value = module.alb.lb_dns_name
}
```

Ta sử dụng Autoscaling Module ở tệp tin `main.tf` như sau.

Copy

Cuối cùng ta khai báo tệp tin `output` cho Root Module. Tạo tệp tin `outputs.tf` ở Root.

Copy

```
output "db_password" {
  value = module.database.config.password
  sensitive = true
}

output "lb_dns_name" {
  value = module.autoscaling.lb_dns
}
```

Ta đã viết code xong, giờ ta chạy câu lệnh `init` và `apply` để tạo hạ tầng nào.

Copy

```
terraform init
```

Copy

```
terraform apply -auto-approve
```

Copy

```
...
Apply complete! Resources: 2 added, 0 changed, 0 destroyed.

Outputs:

db_password = <sensitive>
lb_dns_name = "terraform-series-1259399054.us-west-2.elb.amazonaws.com"
```

Sau khi Terraform chạy xong ta sẽ thấy URL của Load Balancer được in ra Terminal, ta truy cập vào nó.

Copy

```
curl terraform-series-1259399054.us-west-2.elb.amazonaws.com
```

Ta đã tạo được hạ tầng cho một giải pháp Application Load Balancer + Auto Scaling Group + Relational Database Service 😁.

## Content

GitLab Runner là một thành phần quan trọng giúp tự động hóa việc chạy pipeline trong GitLab CI/CD. Nếu bạn đang sử dụng Ubuntu 22.04, bài viết này sẽ hướng dẫn bạn cách cài đặt và cấu hình GitLab Runner một cách chi tiết, đảm bảo hệ thống hoạt động ổn định và hiệu quả.

Trong bài viết này, bạn sẽ học cách:

- Cài đặt GitLab Runner từ nguồn chính thức
- Đăng ký Runner với GitLab

## 1. Cập nhật Chỉ mục Kho Lưu trữ Apt

Để bắt đầu, hãy đăng nhập vào máy chủ của bạn và làm mới bộ nhớ đệm APT cục bộ. Việc này đảm bảo bạn đang lấy thông tin gói mới nhất trước khi tiếp tục.

```
sudo apt update
```

## 2. Tải xuống Script Cài đặt GitLab Runner

GitLab không có sẵn trong các kho lưu trữ mặc định của Ubuntu. Thay vào đó, GitLab cung cấp một script cài đặt riêng để giúp quá trình thiết lập nhanh chóng và dễ dàng. Bạn chỉ cần tải xuống và chạy script cài đặt GitLab Runner như minh họa dưới đây.

```
curl -L "https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.deb.sh" | sudo bash
```

## 3. Cài đặt GitLab Runner trên Ubuntu 24.04

Bây giờ kho lưu trữ đã được thiết lập, hệ thống của bạn đã biết nơi để lấy gói GitLab Runner. Để cài đặt GitLab Runner, chỉ cần sử dụng trình quản lý gói APT với lệnh sau.

```
sudo apt install gitlab-runner
```

## 4. Kiểm tra Trạng thái của GitLab Runner

Theo mặc định, GitLab Runner sẽ tự động khởi động sau khi được cài đặt. Bạn có thể xác minh trạng thái của nó bằng lệnh sau

```
sudo gitlab-runner status
```

## Kết Luận

Vậy là chúng ta đã cài đặt Gitlab Runner trên Ubuntu thành công. Bây giờ bạn hoàn toàn có thể kết nối Gitlab Runner này trên Gitlab để triển khai các CI/CD pipeline.

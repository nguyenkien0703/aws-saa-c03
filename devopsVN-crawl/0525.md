## 19 gói npm độc hại, nhắm đánh cắp credential và khóa crypto

**Một chiến dịch worm chuỗi cung ứng đang hoạt động được phát hiện lợi dụng ít nhất 19 package npm độc hại để đánh cắp token, secrets và khóa tiền mã hóa, đồng thời mở rộng lây nhiễm bằng tài khoản npm và GitHub bị chiếm quyền và nhắm cả các công cụ AI hỗ trợ lập trình.**

![19 gói npm độc hại, nhắm đánh cắp credential và khóa crypto](https://devops.vn/uploads/images/2026/02/346f4d88-25a2-4605-a980-b17ff392de05.webp)

### Tổng quan chiến dịch SANDWORM_MODE

Chiến dịch được đặt tên **SANDWORM_MODE**, mang nhiều dấu hiệu tương tự các đợt Shai-Hulud trước đây: mã độc trong package có thể thu thập thông tin hệ thống, access token, API key và secrets môi trường từ máy dev rồi tự lan bằng danh tính [npm](https://devops.vn/tag/npm) và GitHub bị đánh cắp, theo phân tích trong bài viết này: [phân tích của Socket](https://socket.dev/blog/sandworm-mode-npm-worm-ai-toolchain-poisoning).

Ngoài ra có 4 *sleeper packages* hiện chưa thấy tính năng độc hại: **ethres**, **iru-caches**, **iruchache**, **uudi**.

### Kỹ thuật lây lan và đánh cắp secrets

Rủi ro tăng mạnh vì worm không chỉ lan trong npm mà còn nhúng **GitHub Action độc hại** để hút secrets CI/CD và exfiltrate qua HTTPS, kèm DNS fallback. Payload cũng có routine kiểu kill switch có thể xóa home directory nếu mất quyền truy cập GitHub và npm, nhưng đang tắt mặc định.

### Nhắm AI coding assistants qua MCP

Điểm đáng lo là module **McpInject** nhắm thẳng AI coding assistants bằng cách dựng một MCP server độc hại theo tài liệu [Model Context Protocol](https://modelcontextprotocol.io/docs/getting-started/intro), sau đó đăng ký các tool theo cơ chế [MCP tools](https://modelcontextprotocol.io/legacy/concepts/tools) để nhúng prompt injection đọc file nhạy cảm như ~/.ssh/id_rsa, ~/.ssh/id_ed25519, ~/.aws/credentials, ~/.npmrc và .env rồi gom lại chờ exfiltration. Báo cáo cũng nêu mục tiêu gồm Claude Code, Claude Desktop, Cursor, VS Code Continue và Windsurf, đồng thời thu thập API key của nhiều nhà cung cấp LLM.

Mẫu còn chứa ý đồ né phát hiện bằng engine polymorphic có thể gọi Ollama local với model [DeepSeek Coder](https://deepseekcoder.github.io/) để rewrite mã, dù tính năng này đang tắt trong các package bị phát hiện. Chuỗi tấn công chạy 2 giai đoạn và giai đoạn 2 chỉ kích hoạt sau 48 giờ, kèm jitter theo máy có thể cộng thêm tối đa 48 giờ.

### Khuyến nghị

- Nếu đã cài bất kỳ package nào trong danh sách, cần gỡ ngay.
- **Rotate** npm token, GitHub token và CI secrets.
- Rà soát package.json, lockfiles và .github/workflows để phát hiện thay đổi bất thường.

Cùng thời điểm, các nhóm khác cũng cảnh báo mối đe dọa tương tự: Veracode phân tích buildrunner-dev tại [bài của Veracode](https://www.veracode.com/blog/malicious-npm-package-hiding-in-plain-pixels/), còn JFrog mô tả eslint-verify-plugin trong [bài của JFrog](https://research.jfrog.com/post/three-stages-deep-a-malicious-npm-package/) với agent Mythic như [Poseidon](https://github.com/MythicAgents/poseidon) và [Apfell](https://github.com/MythicAgents/apfell). Checkmarx cũng cảnh báo extension VS Code giả mạo solid281 lợi dụng cơ chế kích hoạt khi ứng dụng khởi động theo tài liệu [onStartupFinished của VS Code](https://code.visualstudio.com/api/references/activation-events#onStartupFinished), chi tiết trong [bài của Checkmarx](https://checkmarx.com/zero-post/solidity-devs-targeted-again-malicious-vs-code-extension-drops-screenconnect-based-remote-access-trojan-rat/).

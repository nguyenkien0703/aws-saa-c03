### Argo CD là gì?

Argo CD là một công cụ mã nguồn mở được thiết kế để hỗ trợ quy trình **GitOps** trong việc triển khai ứng dụng trên **Kubernetes**. Argo CD được áp dụng vào trong bước CD (Continuous Delivery**)** của luồng CI/CD. Đây là bước ta thực thi triển khai ứng dụng lên trên môi trường máy chủ.

Argo CD sử dụng các kho Git làm nguồn thông tin chính để xác định trạng thái ứng dụng cần có, do đó nó được gọi là một công cụ [GitOps](/878c6843d92e45a3af8e2153fd6bb103). Trong series này ta sử dụng Argo CD để triển khai ứng dụng **Microservice Book Info** dưới đây.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/02b3e228-c620-41ca-8f34-c93c85faab3a/argocd-02.drawio_(1)/w=1920,quality=90,fit=scale-down)

Bài đầu tiên ta tìm hiểu cách cài đặt Argo CD và dùng cách đơn giản nhất để triển khai ứng dụng Book Info.

### Cài đặt Argo CD

Tạo namespace cho argocd và cài đặt argocd:

Copy

```
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

Kiểm tra cài đặt thành công:

Copy

```
kubectl get pods -n argocd
```

Copy

**Cài Argo CD CLI** để tương tác với Argo CD, với Mac, Linux hoặc WSL ta chạy câu lệnh sau:

Copy

```
brew install argocd
```

Với Windows tham khảo dường dẫn này [CLI installation documentation](https://argo-cd.readthedocs.io/en/stable/cli_installation/). Truy cập UI của Argo CD:

Copy

```
kubectl port-forward svc/argocd-server -n argocd 8080:443
```

Vì ta chạy localhost nên website báo unsafe, ta bấm proceed to localhost (unsafe):

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/3369a3cf-ccbc-47cd-aea6-60083f778d1c/image/w=1920,quality=90,fit=scale-down)

UI đăng nhập:

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/15c91ea2-4858-4849-b61b-5a4ce0420e9e/image/w=1920,quality=90,fit=scale-down)

Tài khoản mặc định của Argo CD là admin, để lấy password chạy câu lệnh sau:

Copy

```
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
```

UI sau khi đăng nhập vào Argo CD:

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/482fe215-b959-4584-936d-28b14525382a/image/w=1920,quality=90,fit=scale-down)

### Triển khai ứng dụng Book Info

Book Info bao gồm các service sau:

- Products Service
- Details Service
- Reviews Service
- Ratings Service

Để triển khai ứng dụng trên Argo CD ta cần tạo một Application (giải thích chi tiết ở bài sau). Nhấn vào nút **+ New App** ở trên UI.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/0a443ee6-9dd8-44fb-8d6b-df935b1b4c90/image/w=1920,quality=90,fit=scale-down)

Tên ứng dụng ta điền bookinfo, project chọn default, sync policy để Automatic. Các thuộc tính SYNC OPTIONS tạm thời ta để mặc định.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/d1347b26-fcf0-4a34-b895-5554d4e5eac5/image/w=1920,quality=90,fit=scale-down)

Mục **Repository URL** là trường ta kết nối tới Git, nơi chứa tệp tin Kubernetes Resource ta cần triển khai. Các bạn dùng repo <https://github.com/hoalongnatsu/istio-microservice-book-info> của mình. Đây là Public Repo và ai cũng truy cập được, bài sau mình sẽ hướng dẫn cách kết nối tới Private Repo. Phần Path ta điền dấu `.` là thư mục root:

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/baa90e0f-3f93-4141-a1cd-4756f9ce1a56/image/w=1920,quality=90,fit=scale-down)

Mục **Destination** điền vào `https://kubernetes.default.svc` và chọn namespace triển khai, của mình là default.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/fc1e6da9-7828-4f26-825d-a08248423bcf/image/w=1920,quality=90,fit=scale-down)

Bấm nút **Create.**

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/41c1daf7-2192-402d-ac02-8587c46000da/image/w=1920,quality=90,fit=scale-down)

Đợi Argo CD triển khai ứng dụng.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/2b330756-ea21-4125-acc7-daffeb2c4108/image/w=1920,quality=90,fit=scale-down)

Đợi status chuyển sang synced thì ứng dụng bookinfo được triển khai lên môi trường kubernetes thành công. Bấm vào App.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/f14316a6-61d5-4e12-b019-0996deb7373f/image/w=1920,quality=90,fit=scale-down)

Các Service ta định nghĩa bằng tệp tin YAML và lưu trên Git đã được Argo CD triển khai lên Kubernetes. Để kiểm tra ta mở trang Products lên:

Copy

```
kubectl port-forward svc/products 9080:9080
```

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/8d8c70ae-b76e-4a99-a1d4-0284d14682c5/image/w=1920,quality=90,fit=scale-down)

Trong bài này, chúng ta đã triển khai thành công ứng dụng Microservice Book Info bằng cách đơn giản. Bài tiếp theo, chúng ta sẽ đi sâu hơn vào các khái niệm trong Argo CD và áp dụng chúng để triển khai ứng dụng một cách chuyên nghiệp và hiệu quả hơn.

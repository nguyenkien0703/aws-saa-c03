## Sử dụng Remote Backend

Các bạn đọc bài [Terraform Backend](/terraform-series/terraform/bai-7-terraform-backend-backend-trong-terraform-la-gi) để hiểu rõ hơn về ưu và nhược điểm của Remote Backend.

Để sử dụng Remote Backend, ta cần phải tạo tài khoản và đăng nhập vào trong Terraform Cloud.

### Terraform Cloud

Terraform Cloud là một dịch vụ của HashiCorp, nó giúp ta trong việc quản lý resource một cách dễ dàng và bảo mật hơn. Ngoài ra Terraform Cloud còn giúp ta trong việc xây dựng CI/CD cho việc cung cấp hạ tầng rất đơn giản.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/e7c4047e-0c19-4104-b479-b625482a23a0/Untitled/w=1920,quality=90,fit=scale-down)

Terraform Cloud có ba cách sử dụng là:

- Version control workflow.
- CLI-driven workflow.
- API-driven workflow.

Ta sẽ sử dụng CLI-driven Workflow cho Remote Backend, Version Control Workflow cho CI/CD.

### Tạo tài khoản

Đầu tiên để làm việc với Terraform Cloud ta phải tạo tài khoản trước, truy cập vào đường dẫn này [Signup](https://app.terraform.io/signup/account) và tạo tài khoản.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/616b706b-6d82-49ca-ae1f-1c6732c43b71/Untitled/w=1920,quality=90,fit=scale-down)

Sau đó ta đăng nhập vào Terraform Cloud và chọn **Start from scratch**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/9cf196d5-be38-483b-a0dd-d4770c6602bd/Untitled/w=1920,quality=90,fit=scale-down)

Bước tiếp theo ta nhập thông tin về Organization. Sau đó bấm tạo, khi tạo xong ta sẽ thấy UI như bên dưới.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/f0544439-a5a7-443c-97f8-130fe365cf44/Untitled/w=1920,quality=90,fit=scale-down)

Vậy là ta đã chuẩn bị xong, bây giờ ta sẽ làm việc với Remote Backend.

### Sử dụng

Để sử dụng được Remote Backend, ở UI Workspaces ta bấm vào tạo Workspaces và chọn CLI-driven workflow.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/f92b5573-7570-4223-a492-365628eb7159/Untitled/w=1920,quality=90,fit=scale-down)

Sau đó ta nhập vào tên của Workspace và bấm tạo.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/c924156d-aa82-4957-a46d-2b269dafd4ee/Untitled/w=1920,quality=90,fit=scale-down)

Sau khi tạo xong thì Workspace của ta đã sẵn sàng để sử dụng.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/4716bf6f-e5b2-400a-aeb8-fb499d10bdbb/Untitled/w=1920,quality=90,fit=scale-down)

Ta sẽ thấy trạng thái của Workspace là *Waiting for configuration*, tiếp theo ta sẽ cấu hình để Terraform Local kết nối tới Remote Backend.

Kéo xuống ta sẽ thấy phần cấu hình và sử dụng.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/f1bba7f6-ff40-455f-8568-03ad83f4339a/Untitled/w=1920,quality=90,fit=scale-down)

Đầu tiên ta tạo một thư mục và tệp tin `main.tf` với đoạn code.

Copy

Để sử dụng Remote Backend ta cần thêm cấu hình sau vào tệp tin Terraform.

Copy

```
terraform {
  cloud {
    organization = <organization-name>

    workspaces {
      name = <workspace-name>
    }
  }
}
```

Ta sẽ xài `cloud block` với hai thuộc tính là `organization` và `workspaces`. Cập nhật tệp tin `main.tf` với cấu hình của Workspace ta vừa tạo ở trên.

> Sử dụng cloud block với Terraform phiên bản 1.1 trở lên, đối với phiên bản nhỏ hơn 1.1 ta xài remote block.

Copy

```
terraform {
 cloud {
   organization = "HPI"

   workspaces {
     name = "terraform-series-remote-backend"
   }
 }
}

provider "aws" {
 region = "us-west-2"
}

...
```

Tiếp theo ta sẽ chạy câu lệnh `terraform login` để đăng nhập vào Remote Backend. Khi chạy ta sẽ thấy UI như bên dưới.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/b22e3913-4e92-45f3-86ec-0efa85fc69bc/Untitled/w=1920,quality=90,fit=scale-down)

Bấm **Create API token**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/97c21b4f-ff0e-4ef4-8c94-97995cd13e98/Untitled/w=1920,quality=90,fit=scale-down)

Sao chép giá trị và bấm **Done**, quay lại Terminal dán giá trị ta vừa sao chép vào.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/c95f2f51-56f4-4841-bed3-dc4449699317/Untitled/w=1920,quality=90,fit=scale-down)

Nếu giá trị đúng thì Terraform Local của ta đã đăng nhập vào Terraform Cloud thành công.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/3e91faa9-75cb-42e3-8119-85b1dc97fe81/Untitled/w=1920,quality=90,fit=scale-down)

Sau khi đăng nhập xong, ta chạy câu lệnh `init`.

Copy

```
terraform init
```

Copy

Vậy là ta đã cấu hình Remote Backend thành công 😁. Bây giờ ta chạy câu lệnh `terraform plan` thử.

Copy

```
terraform plan
```

Copy

Bạn sẽ thấy nó báo lỗi, vì bây giờ khi ta sử dụng Remote Backend thì toàn bộ cấu hình liên quan tới Credential như là `secret key` và `access key` của AWS ta đều phải cấu hình ở trên Remote Backend. Đây là một điểm mạnh khi ta xài Remote Backend, vì toàn bộ cấu hình liên quan tới Credential ta đều nằm ở một chỗ, không cần phải lưu Credential dưới máy của ta => tăng tính bảo mật.

Để cấu hình Credential ta bấm qua mục **Settings** chọn **Variable sets**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/0c4523ce-53e6-44c3-8e8e-45ad9da3f296/Untitled/w=1920,quality=90,fit=scale-down)

Bấm tạo, đặt tên `variable sets` là AWS Credentials, chọn **Apply to all workspaces in this organization**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/7d9ae08f-d918-4206-b0f7-62d270a65253/Untitled/w=1920,quality=90,fit=scale-down)

Tiếp theo ta tạo biến để lưu hai giá trị `AWS_ACCESS_KEY_ID` và `AWS_SECRET_ACCESS_KEY`. Bấm **Add variable** cho `AWS_ACCESS_KEY_ID`, chọn **Environment variable** và điền vào giá trị, **nhớ chọn biến đó là Sensitive**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/1a7f8bb0-98b4-4369-9c4d-72db6d07a295/Untitled/w=1920,quality=90,fit=scale-down)

Làm tương tự cho `AWS_SECRET_ACCESS_KEY`.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/d664aa3b-51d8-4fab-b05a-2dad5398b5d9/Untitled/w=1920,quality=90,fit=scale-down)

Bấm lưu.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/022056d2-640e-42cc-9b95-b84168015c3b/Untitled/w=1920,quality=90,fit=scale-down)

Giờ ta chạy lại câu lệnh `terraform plan`.

Copy

```
terraform plan
```

Copy

```
Running plan in Terraform Cloud. Output will stream here. Pressing Ctrl-C
will stop streaming the logs, but will not stop the plan running remotely.

Preparing the remote plan...
...

Plan: 1 to add, 0 to change, 0 to destroy.

Changes to Outputs:
  + public_ip = (known after apply)
```

Chạy câu lệnh `apply` nào.

Copy

```
terraform apply
```

Copy

```
...
aws_instance.server: Creating...
aws_instance.server: Still creating... [10s elapsed]
aws_instance.server: Creation complete after 12s [id=i-0839b6f71c5749de4]

Apply complete! Resources: 1 added, 0 changed, 0 destroyed.

Outputs:

public_ip = "34.220.170.155"
```

Khi câu lệnh `apply` chạy xong thì lúc này tệp tin State của ta sẽ được lưu ở trên Terraform Cloud. Ở trên Terraform Cloud, bấm qua phần **State**.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/ac12f7a8-1ea5-4bc5-9b44-c536cc0854d9/Untitled/w=1920,quality=90,fit=scale-down)

Bấm vào **Triggered via CLI** ta sẽ thấy giá trị của tệp tin State. Ngoài ra Terraform Cloud còn hiển thị các resource hiện tại đang có cho ta, giúp ta dễ dàng kiểm tra các resource hiện tại. Bấm qua tab **Overview** kéo xuống phía dưới.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/b5de4053-9d4a-4682-b15f-a7583856b940/Untitled/w=1920,quality=90,fit=scale-down)

**Quan trọng:** lưu ý một điều là khi ta chạy các câu lệnh của Terraform với Remote Backend, thì Terraform Runtime sẽ không chạy ở máy của ta mà sẽ chạy ở Remote Server, và nó sẽ Stream kết quả về máy của ta. Do đó, khi bạn đang chạy mà bạn có bấm `Ctrl + C` để tắt thì nó chỉ tắt Stream, còn Terraform ở Remote Server vẫn chạy bình thường.

![image](https://images.spr.so/cdn-cgi/imagedelivery/j42No7y-dcokJuNgXeA0ig/9a13c6a3-0261-4385-9c1b-e417e1f58e5f/Untitled/w=1920,quality=90,fit=scale-down)
